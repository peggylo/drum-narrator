# 開發前問題檢查

## 目標
在進入下一個開發階段前，識別需要先討論或確認的事項，避免開發中途才發現問題。

## 執行步驟

### 1. 快速定位當前階段
- 查看 PRD、development、solution 相關文件
- 識別「下一步要做的功能/任務」是什麼（一句話描述即可）

### 2. 針對該任務，掃描並識別問題

請檢查以下面向，找出潛在問題：

**需求面**
- 有沒有模糊、不明確的需求描述？
- 有沒有缺少的 edge case 或例外處理說明？
- 使用者流程是否完整？

**技術面**
- 有沒有尚未決定的技術選型？
- 是否有依賴其他尚未完成的功能？
- 資料結構/API 格式是否已定義清楚？

**風險面**
- 有沒有可能與現有功能衝突的地方？
- 是否有效能或安全性的考量需要先討論？
- 有沒有第三方服務/套件的限制需要確認？

### 3. 輸出格式

請用**繁體中文**回覆，格式如下：

```
【即將進行】
用一句話說明下一步要做的任務

【待討論事項】
依重要性排序，列出需要先確認的問題：

1. ❓ [問題描述]
   → 建議：[可能的解法或需要誰來回答]

2. ❓ [問題描述]
   → 建議：[可能的解法或需要誰來回答]

（如有更多問題，繼續列出）

【可直接開始】
如果沒有待討論事項，說明「目前需求明確，可以開始開發」
```

## 輸出要求
- **聚焦問題**：只列出真正需要討論的事項，不要列出已經明確的內容
- **可行動**：每個問題都要有建議的下一步（問誰、查什麼、或提供選項）
- **簡潔**：問題描述控制在 1-2 句話內

## 注意事項
- 如果文件資訊不足以判斷，這本身就是一個問題，請列出「缺少 XX 文件/說明」
- 不要自己腦補答案，這個指令的目的是「提出問題」而非「解決問題」
