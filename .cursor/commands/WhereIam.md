# 專案進度快速檢視

## 目標
快速了解當前專案狀態、已完成工作和下一步行動計畫。

## 執行步驟

### 1. 檢視文件結構
- 查看根目錄中包含 `development`、`prd` 或 `solution` 的 `.md` 檔案
- 查看 `docs/` 資料夾中包含 `development`、`prd` 或 `solution` 的 `.md` 檔案
- 瀏覽其他 `.md` 檔案以獲取補充資訊

### 2. 分析 Git 歷史
- 檢視當前 branch 名稱
- 查看最近 3 筆 commit 記錄(使用 `git log -3 --oneline`)
- 理解最近完成的工作內容

### 3. 整合資訊並輸出

請用**繁體中文**回覆,格式如下:

```
【專案目標】
用 1-2 句話說明本專案的核心目標

【當前分支】
branch 名稱

【最近完成】
根據最近 3 筆 commit 和文件內容,簡述已完成的工作:
- Commit 1 的重點
- Commit 2 的重點  
- Commit 3 的重點

【下一步行動】
根據文件(特別是 development/prd/solution 相關文件)中的規劃,列出 2-3 個下一步要做的事項:
1. 首要任務
2. 次要任務
3. (如有)第三項任務
```

## 輸出要求
- **簡潔為主**: 總字數控制在 160-270 字
- **聚焦當下**: 遠古歷史簡單帶過,重點放在當前狀態和下一步
- **可執行**: 下一步行動要具體明確,讓開發者知道該做什麼
- **整合視角**: 結合 commit 歷史和文件內容,呈現完整脈絡

## 注意事項
- 如果文件中對某些功能標註「已完成」或打勾,請確認與 commit 歷史一致
- 如果找不到相關文件,請明確告知並建議建立專案文件
- 專注於「下一步要做什麼」這個核心問題
