## 《鼓譜 → 講解稿 自動化處理規則 v1.0》

（依照你老師的語風與邏輯打造的可複用規格）

---

### ✦ 0. 規格定位

這套規則用來：

- 從鼓譜 PDF / 圖片自動產生
- 「盲人用語音講解稿」

內容模仿你老師：
- 口訣
- 分小節
- 變化位置
- 特殊小節
- 手法（僅難點）

不加入你老師沒說過的專業鼓理論

其他歌曲也可直接套用

---

### ✦ 1. 譜面分析規則（行 → 小節 → 打擊）

#### 1.1 行結構

以譜面上 橫排 為一行

一行通常包含 8 小節（但不強制）

若碰到 4、小節分行，要以譜面為準

（老師的講解依「行」為單位，因此講解稿也要同步）

#### 1.2 小節分析

每小節拆成：

- Kick（大鼓）
- Snare（小鼓）
- Hi-hat（手）
- Crash
- Tom（Tom1、Tom2、Tom3）
- 空拍 / 休止符

#### 1.3 小節相似性判斷（pattern detection）

兩小節視為「相同」的條件：

- 同樣位置的打擊種類相同
- 無 Crash
- 無過門（Tom）
- 無切分（沒有額外 16 分音符插入）

這用來形成：

- 哪幾小節是重複 pattern
- 哪幾小節是變化。

### ✦ 2. 口訣轉換規則（完全遵照老師語言）

以下是 唯一可用 的口訣詞彙：

鼓組　老師口訣  
大鼓　懂  
小鼓　搭  
小鼓連打　搭搭  
Hi-hat（單擊）　刺  
Hi-hat（連兩下）　刺刺  
Crash　框  
小鼓 + Tom3（過門）　棒  
空拍／延音　嗯  

#### 2.1 一小節＝一整句口訣

（老師的慣例）

例如：

Kick + Snare + Kick ≠「懂搭懂」分開說  
→ 要說成一整句：「懂、搭懂、嗯、搭」

#### 2.2 Crash 必須放在句首

如：

「框刺搭懂」

Crash 不出現在句中

（完全照老師講法）

### ✦ 3. 手法規則（僅依逐字稿）

老師在逐字稿中指出兩種手法情境：

#### 3.1 切分音 →「右、右左」

當小節中出現「刺、刺刺」這種碎音時

自動指定手法：右、右左

若碎音後接小鼓：小鼓為右手

出自錄音：

「第三拍不要用一隻手打，要用右、右左…  
右、右左、右、右。」

#### 3.2 小鼓 + Tom3（棒） → 左手＋右手

「棒」＝小鼓 + Tom3

左手打小鼓

右手打 Tom3

出自錄音：

「左手打小鼓，右手打 Tom3。」

#### 3.3 只在「難小節」講手法

老師從未在一般小節講手法，只有遇到困難段落。

所以：

👉 手法會寫在講解稿內（知識庫）  
但 GPTs 對話時只在難點小節念出來。

---

### ✦ 4. 難度判斷規則（完全來自老師語境）

「難」＝符合以下任一條件（全部來自逐字稿）：

#### 4.1 有切分音

例：第三行第三小節  
老師：「這小節比較難喔。」

→ 小節中出現「刺、刺刺」

#### 4.2 有過門（Fill）

例：第四行第四小節  
老師：「這裡有一個過門，一拍的。」

→ 使用棒（小鼓＋Tom3）

#### 4.3 小節第一拍為 Crash + Kick

例：第三行第一小節  
老師特別提醒：

「第一下要 Crash。」

#### 4.4 老師會給出特殊手法（右、右左）

例：第三行第三小節  
老師說：

「右、右左…這裡比較難。」

因此定義：

👉 難 ≒（切分／過門／Crash起手／特殊手法）任一成立即為難。

### ✦ 5. 講解稿格式（完整模板）

依照老師講課口吻整理成一致的講稿格式：

每行固定格式：

#### 5.1 宣告行數

例如：

「現在講第三行。」

#### 5.2 說明本行共有幾小節

例如：

「這一行有八個小節。」

#### 5.3 指出本行的小節模式

根據 pattern detection 規則：

「前面三小節一樣。」  
「第四小節不同。」

或：

「前兩小節一樣，第三小節不同。」

（不強制三＋一，由 AI 自動分析）

#### 5.4 逐小節口訣（教師版講稿）

格式：

第 1 小節：框刺搭懂、刺刺搭刺  
第 2 小節：懂刺搭懂、刺刺搭刺  
…

注意：

每小節一句完整口訣（遵照老師風格）

難點的小節要標記難度與手法，但不講太多樂理

例：

「第 3 小節：懂刺搭懂、刺、刺刺搭刺（這小節比較難，要用右、右左）。」

#### 5.5 Crash、過門的標示

例：

「第 4 小節有過門，是『懂懂搭懂、刺刺棒』。」

#### 5.6 行結尾總結（老師語氣）

例：

「這行只要注意第三小節。」  
「這行就會了。」

### ✦ 6. GPTs 對話時只念「學生用版本」

講解稿是教師版（詳盡版本）。  
GPTs 對學生講話要簡化：

#### 6.1 若無難點 → 念重點摘要

例如：

「前兩小節一樣，第三小節不同。」

#### 6.2 若有難點 → 只念難的小節

例如：

「這行第三小節比較難，我講一次：  
懂刺搭懂、刺、刺刺搭刺。  
這裡要用右、右左。」
